# CLAUDE.md — Правила работы с проектом Quiz Electro

## О проекте

Интерактивный веб-квиз для частного электрика из Гомеля. Инструмент лидогенерации: пользователь отвечает на вопросы — получает диагноз и оставляет телефон.

---

## Общие правила

- **Один файл на страницу.** Каждая страница — самодостаточный HTML-файл (HTML + CSS + JS внутри). Никаких внешних библиотек (jQuery, Bootstrap и т.д.). Только Google Fonts — допустимо.
- **Перед созданием нового файла** — сначала описать план: что будет в файле, какова структура, какие секции. Дождаться подтверждения, затем писать код.
- **Перед редактированием существующего файла** — кратко описать, что и зачем меняем.

- Акцентный оранжевый — только для кнопок, выделений, активных состояний.

---

## Адаптивность

- Mobile-first. Всё должно корректно работать с **320px**.
- Контрольная точка: `@media (max-width: 480px)`.
- Кнопки — полная ширина на мобильных (`width: 100%`).
- Никаких горизонтальных скроллов.

---

## Стиль текста (контент, подписи, сообщения)

- Простой, дружеский, без канцелярита.
- Обращение на «вы» (с маленькой буквы, как принято в русском языке).
- Без клише: «осуществить», «в рамках», «является», «данный». Писать по-человечески.
- Ошибки валидации — конкретные и вежливые: «Введите номер телефона (минимум 9 цифр)», а не «Ошибка валидации поля».

## Технические требования

### Стек
- HTML5, CSS3 (flex/grid), JavaScript ES6+ (чистый, без фреймворков).
- Данные квиза хранятся в JS-объекте в памяти (не в localStorage, если явно не попросят).

### Квиз (quiz.html)
- Пошаговый интерфейс: один вопрос на экране.
- Прогресс-бар: «Вопрос X из 8» + визуальная полоска.
- Кнопки: «Назад» / «Далее». На последнем вопросе — «Показать результат».
- Тип вопроса: `radio` (одиночный выбор) или `checkbox` (множественный).
- Ответы хранятся в объекте до финального нажатия; пересчёт — только при нажатии «Показать результат».
- После результата — кнопка «Пройти заново» (сброс всех ответов, возврат к вопросу 1).
- Анимации появления вопросов: `fade-in` (CSS transition, ~300ms).

### Форма контактов
- Поля: имя (не пустое, ≥ 2 букв) + телефон (≥ 9 цифр).
- Валидация при сабмите, ошибки показывать под соответствующим полем.
- После успешной отправки:
  - Показать сообщение: «Спасибо! Я перезвоню в течение часа. А пока скачайте чек-лист».
  - Вывести в `console.log` объект `{ name, phone, answers }`.
  - Кнопка отправки блокируется на 5 секунд (защита от двойного клика).
- Ссылка на чек-лист: атрибут `href="#"` + `download` (заглушка).

### Логика определения типа клиента
- 5 типов: Аварийный, Новая прокладка, Только щиток, Ограниченный бюджет, Премиум.
- Приоритет проверки: сверху вниз (Тип 1 → Тип 5).
- Если ни одно условие не сработало — показывать Тип 1 как универсальный.
- Логику определения типа оформить отдельной функцией `determineType(answers)` с комментариями к каждой ветке.

### Комментарии в коде
- Логику квиза и определения типа — комментировать подробно.
- CSS и тривиальный JS — без лишних комментариев.
